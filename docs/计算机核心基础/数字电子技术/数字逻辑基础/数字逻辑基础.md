# 数字逻辑基础

## 进制

### 进制基数

- 二进制(bin)：只有0和1两个数字，是计算机最常用的进制
- 八进制(oct)：早期为了方便推算出2进制的进制数，3个2进制为一个八进制，应用于Linux的文件权限
- 十进制(dec)：人类所使用的进制基数
- 十六进制(hex)：易于人类理解和处理，应用于内存地址表示、ipv6地址和mac地址表示

### 进制转换

- 其他进制转换二进制：除2取余数，除到除数为0，再倒叙排列
- 二进制转换成八进制：将3个二进制看成一位
- 二进制转换成十进制：前面一位数是后面一位数的2倍，最开始的数是1
- 二进制转换成十六进制：将4个二进制看成一位

## 有符号表示

> 用补码进行存储，方便计算，目的：节省减法器开销

- 原码：进制转换后的二进制码，共八位，最高位1代表负数、0代表正数
- 反码：如果是负数将原码所有数取反（除了最高位），正数则不变
- 补码：如果是负数在反码基础上+1，正数则不变

## 字符编码

> 字符和二进制数的制定规则，以下是常用字符编码

- gbk：中国的字符编码(仅能够识别常用的简体字和英文)
- ascii：早期互联网的通用编码(7位二进制表示字符)
- Unicode：万国码
  - utf-8：目前通用的字符编码，兼容ascll码，以7位二进制表示字符，最高位一直为0
  - utf-16：utf-8的进阶版(16位二进制表示字符)
  - utf-32：utf-16的进阶版(32位二级制表示字符)
