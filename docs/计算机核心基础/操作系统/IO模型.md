# IO模型

## 网络io

- recv：阻塞明显
- recvfrom
- send：阻塞不明显
- accept：等待连接对象

## 阻塞IO

> 一直阻塞直到，返回执行的结果

![](/IO模型/阻塞IO模型-导出.png)

## 非阻塞IO

> 不停的向系统内核要数据，不会受wait for data阶段的影响，仅可能多用CPU，缺点：占用CPU资源大

![](/IO模型/非阻塞IO模型-导出.png)

## IO多路复用

> 使用监管机制，如果有连接就获取连接，进行处理连接

![](/IO模型/IO多路复用-导出.png)

## 异步IO

> 提交任务给操作系统，操作系统监管这个任务执行结果，自动将结果返回

![](/IO模型/异步IO模型-导出.png)
