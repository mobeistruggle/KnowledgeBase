# 用户管理

------

## 切换用户

```shell
# 切换到指定用户
su  - '用户名'

# 切换到root用户
su
```

### 修改用户/用户组

> 修改文件/目录的用户或用户组

```shell
# 修改用户
chown '用户' '文件/目录'

# 修改用户组
chown :'用户组' '文件/目录'

# 修改指定目录及其内容的用户/用户组
chown -R '用户':'用户组' '文件/目录'
```



```shell
# 修改权限, 两者一样
vi /etc/sudoers
visudo

# 不用输入密码就可以执行sudo命令
'用户名' ALL=(ALL)		NOPASSWORD:ALL

# 当前命令，以root用户权限执行
sudo '命令'
```

### 用户基本信息文件

```shell
# 用户信息存储在 /etc/passwd 文件,使用cat命令查看用户信息的全部内容
cat /etc/passwd
# 用户文件信息解析
root：登录系统的用户名
x：密码占位符，密码具体位置不在这
0：Uid,用户的身份证号(0是超级管理员用户、1~999是系统用户、1000+是普通用户)
0：Gid,用户的group组号(Gid和Uid是一样的)
root：描述
/root：家目录,登录系统时,所在的目录
/bin/bash：登录shell,命令解释器

# 用户信息的格式
root:x:0:0:描述:/root:/bin/bash						# 超级用户
mysql:x:10:10:系统自动创建的用户:/app:命令解释器			# 系统用户
mobei:x:1001:1001:超级用户创建的用户:/home:命令解释器		# 普通用户
```

### 用户密码信息文件

```shell
# 用户密码存放的位置
	/etc/shodow

# 用户密码信息解析($1是md5加密算法、$5是sha-256加密算法、$6是sha-512加密算法)
root：用户名
$1加密后的密码：用户密码
19599：最后修改时间(从1970年开始算起)
7：修改密码时间间隔(修改密码过7天才能在修改)
9999：密码有效期(密码有效9999天)
7：警告密码过期时间(密码倒数7天后系统发出警告)
10：密码超过过期时间多少天不能使用(密码过期10天后不能使用)
4：密码失效时间(密码4天后失效不能登录系统)
保留：占位子

# 用户密码信息文件格式
root:$1加密算法:最后一次修改时间:密码多少天后可以在此修改:密码有效期:密码过期前系统警告:密码过期天数不能使用:密码失效时间:保留

```

### 用户组信息文件

```shell
# 用户组信息文件存放位置
	/etc/group

# 用户组信息解析
root：用户组名
x：用户组的密码
99：用户组id(类型用户id)
root：组成员

# 用户组信息文件格式
root:x:99:用户组成员

```

### 用户信息管理指令

```shell
# 查看当前用户
whoami

# 创建用户,并创建与该用户名同名的组并加入
useradd '用户名'

# 创建用户并修改用户id，id号需为1000以上
useradd '用户名' -u 'id号'

# 创建用户并指定该用户加入的组名
useradd '用户名' -g '组名'

# 禁止指定用户登录shell
useradd -s /sbin/nologin '用户名'

# 查看指定用户信息，不写用户名默认查看当前用户
id '用户名'

# 删除指定用户
userdel '用户名'

# 删除指定用户的所有信息（home目录）
userdel -r '用户名'

# 修改指定用户的密码
passwd '用户名'

# 修改指定用户所在组
usermod -aG '用户组' '用户名'

# 查看当前系统所有用户的信息，信息格式：用户名:密码(用x代替):用户id:组id:描述信息(默认空):用户目录位置:所用终端位置
getent passwd
```

### 用户组信息管理指令
